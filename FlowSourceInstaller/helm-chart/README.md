<h1>Flowsource Charts Repo</h1>

Helm chart for Flowsource. 
The chart creates the required Kubernetes config maps, secrets and deployment files required for Flowsource. 
It does not create any database or other ecosystem components suuch as JIRA, Sonarqube, etc.

Postgres database required should be created separately as either a Managed Database on the cloud or a Postgres database running as a container within K8s.

<h2>Hot to use Flowsource Helm chart</h2>
Flowsource requires certain configuration files such as app-config.yaml and all-configurations.yaml. These contain customized configuration values and need to be provided as values file at the time of Helm install. Example files for these can be found in the docs folder.

Once the chart is downloaded locally, you can install the chart along with the required configuration files using the command below:
```
helm install --upgrade -f my-all-configurations-values.yaml -f my-app-config-values.yaml -f my-values.yaml  my-flowsource-deployment .
```

To enable the Kubernetes plugin for Azure , we need to add the cluster authourity  data certificate in the ca-certs.yaml file that can be generated by

	1)Copy the base64 encoded authority data(caData specified in app-config.yaml) to caData.base64

	2)Run the below commands in shell and copy the content of caData.pem to the ca-certs.yaml

	  base64 -d caData.base64 > caData.crt

	  openssl x509 -in caData.crt -out caData.pem -outform PEM 

use below command with ca-certs.yaml to install the charts
```
helm install --upgrade -f my-all-configurations-values.yaml -f my-app-config-values.yaml -f my-values.yaml -f ca-certs.yaml my-flowsource-deployment .
```
